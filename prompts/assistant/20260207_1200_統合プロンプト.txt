# 目的
カレンダー横に女性キャラクターを配置し、日付クリック時にその日のTODO内容を吹き出しへ表示するUIに変更する。
キャラクターは 4列×3行のスプライトシート（1264×848px）からパラパラ漫画のようにアニメーション表示する。

# 前提
- 既存のカレンダー日付クリックでTODO一覧が表示される挙動を活用する。
- 既存のデータ取得・日付選択ロジックは維持し、表示先のみ吹き出しへ切り替える。
- 画像は `src/main/resources/static/img/assistant-sprite.png` を使用する。
- スプライトは 4列×3行（合計12フレーム）、全体サイズ 1264×848px。

# 画面要件
- カレンダーの右（または横）に女性キャラクター表示枠を配置する。
- キャラクターの左側に吹き出しを配置し、吹き出し内に選択日のTODO内容を表示する。
- 吹き出しは空欄にならないよう、未選択時は「日付を選択してください」を表示する。
- その日のTODOが0件の場合は「この日のTODOはありません」を表示する。
- TODOが複数件ある場合は、タイトルを改行（または箇条書き）で表示する。
- 既存のカレンダーUI（モーダル/一覧）を残す場合は、吹き出し表示と同期させる。
- スマホ幅ではカレンダーの下にキャラクターと吹き出しが並ぶようにレスポンシブ対応する。

# 実装指示（DOM/JS）
- カレンダー日付クリックのイベントで、取得済みのTODO一覧を吹き出し用の表示エリアへ流し込む。
- 吹き出し表示エリアはIDやクラスを付与してDOM更新対象にする。
- 既存のDOM更新処理がある場合は再利用し、二重管理を避ける。
- クリック時の内容更新に簡単なフェード/切替アニメーションを付与してもよい（既存のアニメーション方針に合わせる）。

# 実装指示（スプライト/CSS方式）
- `assistant-sprite.png` を `background-image` で表示する。
- 1フレームサイズは「全体サイズ ÷ 列/行」で算出する。
  - 例: 幅 = 1264 / 4 = 316px
  - 高さ = 848 / 3 = 282.666...px
  - 実際のフレームサイズに合わせて端数を調整（トリミング/余白がある場合は実測を優先）。
- `background-size` は全体サイズ（1264px 848px）で設定する。
- 2つのアニメーションで X/Y を同期させ、4列×3行を順に再生する。
  - X方向（列）: 4コマを `steps(4)` で移動
  - Y方向（行）: 3コマを `steps(3)` で移動
  - XはYの1/3周期でループさせ、合計12フレームを再生する

# CSS例（4列×3行）
```
.assistant-sprite {
  width: 316px;   /* 実際のフレーム幅に合わせる */
  height: 283px;  /* 実際のフレーム高さに合わせる */
  background-image: url('/img/assistant-sprite.png');
  background-repeat: no-repeat;
  background-size: 1264px 848px;
  animation:
    sprite-x 0.4s steps(4) infinite,
    sprite-y 1.2s steps(3) infinite;
}

@keyframes sprite-x {
  from { background-position-x: 0px; }
  to   { background-position-x: -948px; } /* (316px * (4-1)) */
}

@keyframes sprite-y {
  from { background-position-y: 0px; }
  to   { background-position-y: -565px; } /* (283px * (3-1)) 端数は実測に合わせる */
}
```

# 実装指示（JS方式・必要時）
- 画像サイズが可変/レスポンシブでCSS制御が難しい場合のみJSで対応する。
- `setInterval` でフレーム番号を進め、列/行を計算して `backgroundPosition` を更新する。
- フレーム数は12固定で定数管理する（row=3, col=4）。

# 吹き出しCSS
- CSSで吹き出し形状（角丸＋しっぽ）を実装する。

# 期待する結果
- 日付クリックで吹き出しにその日のTODOが即時表示される。
- 未選択・0件のケースでも必ずメッセージが表示される。
- PC/スマホ双方で崩れずに見える。
- 4×3スプライトから切り替わる形で、キャラクターがパラパラ漫画のように動く。
